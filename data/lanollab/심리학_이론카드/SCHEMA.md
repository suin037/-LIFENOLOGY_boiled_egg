# 심리학 이론카드 표준 스키마 v1

> Parallel Me · 심리학 자료(RAG 기반 질의·응답) 파트
> 목적: 심리학 논문·이론을 **검색 가능한 카드 단위**로 표준화하여, 레이어 2의 지표 점수·감정 신호를 쿼리로 받아 관련 이론을 retrieve하고, 레이어 3(Claude 내러티브 생성)의 근거로 주입한다.

---

## 0. 설계 원칙

1. **1카드 = 1개념.** 논문 1편을 통으로 넣지 않는다. "문제중심 대처", "정서중심 대처"처럼 검색·인용 단위가 되는 세부 개념 하나당 카드 한 장으로 쪼갠다. (검색 정확도의 핵심)
2. **쿼리와 매칭될 필드를 반드시 둔다.** Parallel Me의 쿼리는 사용자가 쓰는 자연어가 아니라 **레이어 2의 3지표 점수 + 일기/체크인 감정 키워드**다. 따라서 카드에 `적용상황(applicable_situations)`을 구조화해 넣어, "삶의 질 낮음 + 후회 키워드 → 이 카드" 매칭이 가능하게 한다.
3. **생성 근거로 바로 쓰이게 한다.** `개입방법(interventions)`은 레이어 3의 "오늘 할 수 있는 행동 제안"으로 직행하므로, 추상론이 아니라 실행 가능한 행동으로 적는다.
4. **출처 없는 문장은 넣지 않는다.** RAG의 존재 이유가 "검증된 근거"이므로 모든 카드에 출처를 필수로 둔다.

## 1. 필드 정의

| 필드 | 타입 | 필수 | 설명 |
|---|---|---|---|
| `card_id` | string | ✅ | 전역 고유 ID. 규칙: `{이론약칭}_{개념약칭}_{일련번호}` 예: `coping_problem_001` |
| `theory_ko` / `theory_en` | string | ✅ | 이론명(국문/영문) |
| `theorist` | string | ✅ | 제안자 |
| `year` | integer | ✅ | 발표 연도 |
| `concept_ko` | string | ✅ | 이 카드가 다루는 세부 개념명 |
| `summary` | string | ✅ | 핵심 개념 요약(2~4문장). **임베딩(검색) 대상의 중심 텍스트** |
| `applicable_situations` | object | ✅ | 이 카드를 언제 꺼낼지. 아래 1-1 참조 |
| `interventions` | string[] | ✅ | 개입방법 = "오늘 할 수 있는 행동" 후보. 각 항목은 실행 가능한 한 문장 |
| `narrative_guidance` | string | ⬜ | 레이어 3이 서사 생성 시 참고할 톤·메시지 가이드 |
| `source` | object | ✅ | 출처. 아래 1-2 참조 |
| `tags` | string[] | ⬜ | 자유 태그(검색 보조) |
| `embedding_text` | string | ⬜ | 검색용 통합 텍스트(미기입 시 `concept_ko + summary`로 자동 생성) |
| `version` | string | ✅ | 카드 스키마 버전. 현재 `v1` |

### 1-1. `applicable_situations` (매칭용 — 가장 중요)

| 하위 필드 | 타입 | 설명 |
|---|---|---|
| `indicator` | enum[] | 관련 지표. `경제적_안정도` / `성장_가능성` / `삶의_질` 중 (레이어 2·결과화면·데이터셋 공용 3지표) |
| `direction` | enum | 이 카드를 꺼낼 지표 방향. `낮을수록_적용` / `높을수록_적용`. **낮음/중간/높음 임계값은 카드가 아니라 로더 설정에 둔다** — 아래 주의 참조 |
| `emotion_keywords` | string[] | 매칭 감정 키워드. 예: `["불안","후회","무력감"]` |
| `decision_types` | string[] | 관련 선택 유형(선택). 예: `["이직","진학","창업"]` |

> **3지표 확정:** 세 번째 지표의 정본은 **`삶의_질`**이다(개요 상단 '핵심 아이디어'의 "건강 리스크"는 초기 표현이며, 파이프라인·결과화면·데이터셋 표가 모두 `삶의_질`로 통일 — 건강은 삶의 질 안으로 흡수됨). 카드 매핑은 `경제적_안정도 / 성장_가능성 / 삶의_질` 세 이름만 사용한다.

> **조건 임계값을 카드에 넣지 않는 이유(중요):** 레이어 2(XGBoost)가 지표를 연속 점수로 줄지, 이미 낮음/중간/높음으로 버킷팅해 줄지 아직 미확정이다. 그래서 카드에는 **지표 이름(확정)과 매칭 방향(`direction`)만** 두고, 실제 임계값·라벨 매핑은 **로더 설정(`indicator_thresholds`)**으로 분리한다. 예: `{"삶의_질": {"낮음": "<0.33", "중간": "0.33~0.66", "높음": ">0.66"}}`. 예측 팀이 출력 포맷을 바꿔도 카드는 손대지 않고 로더 설정만 바꾸면 되며, "매칭 안 됨(죽은 데이터)" 문제를 조건 포맷 불확정성으로부터 분리한다.

> **ChromaDB 적재 시 주의:** ChromaDB 메타데이터 값은 스칼라(str/int/float/bool)만 허용된다. `emotion_keywords`·`indicator` 같은 리스트는 적재 단계에서 `"불안,후회,무력감"`처럼 콤마 문자열로 평탄화(flatten)하거나, 별도 매칭 로직에서 처리한다. 리치 카드(JSON)는 원본(source of truth)으로 유지하고, 로더가 ChromaDB용으로 변환한다.

### 1-2. `source`

| 하위 필드 | 타입 | 설명 |
|---|---|---|
| `citation` | string | 정식 인용문(APA 권장) |
| `doi_or_url` | string | DOI 또는 URL(있으면) |
| `evidence_level` | enum | `이론` / `실증연구` / `메타분석` / `척도` |

## 2. 검색 → 생성 흐름에서 카드가 쓰이는 방식

```
[레이어 2 출력]  3지표 점수 + (일기/체크인) 감정 키워드
        ↓  쿼리 자동 생성
   예) 삶의_질=0.21(로더가 '낮음'으로 버킷팅) → direction=낮을수록_적용 카드 대상, emotion=[후회, 불안]
        ↓  ChromaDB 검색 (psych_theory 컬렉션)
   → summary 임베딩 유사도 + applicable_situations 메타 필터
        ↓  top-k 카드 retrieve
[레이어 3 / Claude]  summary·interventions·source를 근거로
   → 서사 시나리오 + "오늘 할 수 있는 행동 제안" 생성 (출처 인용 강제)
```

> 통계 RAG(`stats_kr` 컬렉션)와 **컬렉션을 분리**한다. 심리학 카드는 `psych_theory` 컬렉션에 적재.

## 3. 파일 규칙

- 이론(논문) 1편 = JSON 파일 1개. 파일 안에 그 이론의 카드들을 배열로 담는다.
- 파일명: `cards_{이론약칭}_v{n}.json` 예: `cards_coping_v1.json`
- 최상위 구조:

```json
{
  "theory_ko": "…",
  "theory_en": "…",
  "schema_version": "v1",
  "cards": [ { …카드1… }, { …카드2… } ]
}
```
